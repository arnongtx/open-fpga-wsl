# 🛠️ open-fpga-wsl

คลังเก็บสคริปต์ติดตั้งและชุดเครื่องมือเปิด (Open Source FPGA Toolchain) บนระบบปฏิบัติการ WSL (Ubuntu) แบบรวดเร็วผ่านเครื่องมือจัดการแพ็กเกจ `apt` รองรับการออกแบบด้วยภาษา VHDL และชิป FPGA ยอดนิยมทั้งตระกูล **Lattice iCE40** และ **Lattice ECP5** โดยไม่ต้องเสียเวลาคอมไพล์จากซอร์สโค้ดเอง

🌐 **Repository:** [open-fpga-wsl](https://github.com/arnongtx/open-fpga-wsl)
📜 **License:** [MIT License](LICENSE)

---

## 📦 เครื่องมือที่ติดตั้งภายในระบบ (Included Tools)

*   **Yosys:** เครื่องมือสังเคราะห์วงจรลอจิก (Logic Synthesis)
*   **GHDL:** ตัวจำลองสถานะและคอมไพล์ภาษา VHDL (VHDL Simulation)
*   **nextpnr-ice40 / nextpnr-ecp5:** เครื่องมือจัดวางอุปกรณ์และหาเส้นทางพิน (Place & Route)
*   **GTKWave:** โปรแกรมแสดงกราฟคลื่นสัญญาณวงจรดิจิทัล (Waveform Viewer)
*   **openFPGALoader / dfu-util:** เครื่องมือดาวน์โหลดและแฟลชไฟล์บิตสตรีมลงบอร์ด FPGA ผ่านพอร์ต USB
*   **netlistsvg:** เครื่องมือสร้างภาพผังวงจรดิจิทัล (Schematic Generator) จากไฟล์รูปแบบ JSON
*   **Make / CMake / GCC / G++:** ระบบควบคุมการบิวด์และชุดคอมไพเลอร์พัฒนาข้ามภาษาภาษา C/C++
*   **GDB:** ตัวตรวจสอบระบบและไล่บั๊กส่องค่าสัญญาณภายในคอมพิวเตอร์โฮสต์ (GNU Debugger)

## 🚀 ขั้นตอนการติดตั้งด่วน (Quick Installation)

เปิดโปรแกรมควบคุมระบบ (WSL Terminal) บน Ubuntu จากนั้นคัดลอกกลุ่มคำสั่งด้านล่างนี้ไปวางเพื่อทำการคัดลอกคลังข้อมูลและสั่งรันสคริปต์ติดตั้งระบบโดยอัตโนมัติ:

```bash
# 1. คัดลอกคลังข้อมูลลงมาในเครื่องคอมพิวเตอร์ของคุณ
git clone https://github.com/arnongtx/open-fpga-wsl

# 2. เข้าไปยังโฟลเดอร์โปรเจกต์
cd open-fpga-wsl

# 3. กำหนดสิทธิ์ให้สคริปต์สามารถรันซอฟต์แวร์ได้
chmod +x setup.sh

# 4. เริ่มต้นกระบวนการติดตั้งระบบอัตโนมัติ
./setup.sh
```

## 📂 โครงสร้างพื้นที่ทำงานและการบิวด์แยกส่วน (Workspace Structure)

หลังจากสคริปต์ทำงานเสร็จสิ้น ระบบจะสร้างโฟลเดอร์และไฟล์ตัวอย่างสำหรับทดสอบ โดยระบบใช้ **Isolate Build Pattern** ไฟล์ที่เกิดจากการคอมไพล์ทั้งหมดจะถูกโยกแยกไปเก็บไว้ในโฟลเดอร์เดี่ยวชื่อ `build/` เพื่อรักษาความสะอาดของซอร์สโค้ดหลัก:

```text
fpga-workspace/

│   │   ├── blinky.vhd      # ซอร์สโค้ดไฟกระพริบภาษา VHDL หลัก
│   │   ├── blinky_tb.vhd   # ไฟล์จำลองสถานะลอจิก (Testbench)
│   │   ├── sim_core.cpp    # ไฟล์ฟังก์ชันภาษา C++ เสริมร่วมระบบประมวลผล
│   │   ├── ice40.pcf       # ไฟล์กำหนดพินฮาร์ดแวร์ (.pcf)
│   │   ├── Makefile        # ไฟล์คำสั่งลัดสำหรับ iCE40 Flow
│   │   └── build/          # โฟลเดอร์เก็บไฟล์จากการบิวด์ทั้งหมด (สร้างขึ้นหลังรัน make)
│   └── ecp5_vhdl_blinky/   # โปรเจกต์ตัวอย่างสำหรับ Lattice ECP5
│       ├── blinky.vhd      # ซอร์สโค้ดไฟกระพริบภาษา VHDL หลัก

│       ├── sim_core.cpp    # ไฟล์ฟังก์ชันภาษา C++ เสริมร่วมระบบประมวลผล
│       ├── ecp5.lpf        # ไฟล์กำหนดพินฮาร์ดแวร์ (.lpf)

└── slides/

```

## 🔗 การทำ Co-Simulation ด้วย VHPIDirect (VHDL + C++ Hybrid Flow)

คลังโครงสร้างระบบนี้รองรับคุณสมบัติ **VHPIDirect** ขั้นสูง เพื่อผูกและแชร์คำสั่งทำงานร่วมกันระหว่างฮาร์ดแวร์จำลองภาษา VHDL กับโปรแกรมระบบเปิดฝั่งสถาปัตยกรรมภาษา **C++ (`.cpp`)** โดยระบบตัวอย่างได้ใส่ตัวอย่างโค้ด `sim_core.cpp` (ที่คลุมด้วยบล็อกสิทธิ์แบบ `extern "C"`) เพื่อพิมพ์ส่งพาสทรูบันทึก Log ดึงค่าจาก Register บนชิป FPGA ออกมาแสดงผลทางหน้าจอ WSL Terminal ได้ทันทีเมื่อสั่งรันกระบวนการจำลอง:

```bash
make sim
```

### การลิงก์ข้ามภาษาระดับตัวรันใน Makefile:
1. ระบบเรียกคอมไพเลอร์รันคำสั่ง `g++ -g -c sim_core.cpp` เพื่อแปลงซอร์สโค้ด C++ ออกมาเป็นออบเจกต์ไบนารีฝังข้อมูลดีบั๊กไว้ที่ไฟล์ `build/sim_core.o`
2. ระบบ GHDL ดึงชุดตัวออบเจกต์ดังกล่าวประสานการรวมฟังก์ชันประมวลผลผ่านคำสั่งแนบลิงก์เกอร์ของไลบรารีสแตนดาร์ด C++ ด้วยแฟล็กออปชัน `-Wl,sim_core.o -Wl,-lstdc++` ส่งผลให้สามารถจำลองวงจรร่วมกันแบบไฮบริดข้ามภาษาได้อย่างแม่นยำ

## ⚙️ ขั้นตอนการติดตั้งและเตรียมระบบ USBIPD บน Windows (Prerequisites)

เนื่องจากสคริปต์ `openFPGALoader` รันอยู่บนฝั่ง Linux (WSL) แต่ตัวบอร์ด FPGA เสียบเข้ากับฝั่ง Windows คุณจึงจำเป็นต้องเตรียมระบบฝั่ง Windows ล่วงหน้าด้วยขั้นตอนดังนี้:

### 1. อัปเดตระบบ WSL ให้พร้อมรองรับ USB Pass-through
เปิด PowerShell (สิทธิ์ทั่วไป) บน Windows แล้วสั่งอัปเดตเคอร์เนลให้เรียบร้อย:
```powershell
wsl --shutdown
wsl --update
```

### 2. ติดตั้งเครื่องมือ usbipd-win บน Windows Host
เปิด PowerShell (สิทธิ์ทั่วไป) แล้วพิมพ์คำสั่งนี้เพื่อดาวน์โหลดและติดตั้งตัวจัดการอัตโนมัติ:
```powershell
winget install --interactive --exact dorssel.usbipd-win
```

## ⚠️ การแก้ปัญหาเมื่อบอร์ดไม่แสดงในรายการ (Troubleshooting)

หากเปิด PowerShell แล้วพิมพ์ `usbipd list` แต่ระบบมองเห็นแต่ฮาร์ดแวร์พื้นฐานของระบบ (ไม่พบบอร์ด FPGA) เช่น:
```text
Connected:
BUSID  VID:PID    DEVICE                                                        STATE
1-4    0bda:c811  Realtek 8811CU Wireless LAN 802.11ac USB NIC                  Not shared
1-6    046d:c077  USB Input Device                                              Not shared
```
**สาเหตุเกิดจาก Windows ยังไม่ได้ลงไดรเวอร์ที่เข้ากันได้กับชุดเครื่องมือ Open Source** ให้แก้ไขตามขั้นตอนนี้:

1. **ตรวจสอบฮาร์ดแวร์ก่อน:** ตรวจสอบว่าสาย USB ที่เชื่อมต่อรองรับการส่งผ่านข้อมูล (Data Cable) ไม่ใช่สายชาร์จไฟธรรมดา และเสียบเข้าช่องโปรแกรม (Data/JTAG) บนบอร์ด
2. **ติดตั้ง Driver ด้วย Zadig (สำหรับชิป FTDI/Interface):**
   * ดาวน์โหลดโปรแกรมเปิดสิทธิ์ [Zadig](https://zadig.akeo.ie/) มาเปิดใช้งานบน Windows
   * คลิกที่เมนู **Options** เลือก **List All Devices**
   * แถบด้านบนให้เลือกตัวอุปกรณ์ของบอร์ด FPGA (เช่น `FT2232H`, `Dual RS232` หรือ `USB Serial`)
   * ในช่องไดรเวอร์ปลายทาง (แถบฝั่งขวาที่มีลูกศรชี้มา) ให้กดสลับเลือกเป็น **WinUSB** หรือ **libusbK**
   * กดปุ่ม **Replace Driver** หรือ **Install Driver** แล้วรอจนระบบประมวลผลเสร็จสิ้น
3. **ตรวจสอบอีกครั้ง:** เรียกคำสั่ง `usbipd list` บน PowerShell รอบนี้จะตรวจพบอุปกรณ์ตัวใหม่เพิ่มขึ้นมาในรายการพร้อมตัวเลข `BUSID`

## 🔌 การเชื่อมต่อบอร์ดฮาร์ดแวร์จริงเข้าสู่ระบบ WSL

เมื่อในรายการคำสั่งเจอตัวบอร์ด FPGA โผล่ขึ้นมาแล้ว ให้ดำเนินการเชื่อมต่อระบบผ่าน PowerShell และ WSL ตามขั้นตอนดังนี้:

1. เปิดโปรแกรม PowerShell ด้วยสิทธิ์ **Run as Administrator** (ทำเฉพาะครั้งแรกเพื่อเปิดสิทธิ์แชร์พอร์ต)
2. สั่งผูกสิทธิ์เข้ากับระบบเพื่อเปิดแชร์พอร์ตอุปกรณ์ (แทนที่ `<BUSID>` ด้วยรหัสตัวเลขพอร์ตที่เจอ เช่น `1-10`):
   ```powershell
   usbipd bind --busid <BUSID>
   ```
3. สั่งส่งตัวอุปกรณ์ผ่านโครงข่ายพาสทรูไปยังระบบปฏิบัติการ Linux (คำสั่งนี้ใช้สิทธิ์ธรรมดารันได้ทุกครั้งที่เสียบใช้งาน):
   ```powershell
   usbipd attach --wsl --busid <BUSID>
   ```
4. ย้อนกลับมาที่ WSL Terminal ของคุณ จากนั้นพิมพ์คำสั่งเพื่อแฟลชไฟล์ลงบอร์ดจริงผ่านเครื่องมือ `openFPGALoader`:
   ```bash
   make prog
   ```

## 🎛️ การส่งพารามิเตอร์ควบคุมผ่านคำสั่ง Terminal (Variable Overrides)

ระบบออกแบบมาให้รองรับการเปลี่ยนขนาดเกต (LUTs) ชนิดตัวถังชิป และเวลาหยุดจำลองสถานะได้โดยตรงผ่านหน้าต่างคำสั่งตอนรัน โดยใช้โครงสร้างแบบ **Conditional Assignment (`?=`)** ทำให้คุณไม่ต้องเปิดหน้าไฟล์เข้าไปแก้ไขซอร์สโค้ดภายใน:

### 🧊 ตัวอย่างการกำหนดพารามิเตอร์ของชิป iCE40
*   **ปรับแต่งข้ามรุ่นชิป (เช่น ชิป 1K ตัวถัง TQ144):**
    ```bash
    make DEVICE=1k PACKAGE=tq144
    ```

### 💎 ตัวอย่างการกำหนดพารามิเตอร์ของชิป ECP5
*   **ปรับแต่งข้ามรุ่นชิป (เช่น ชิป 12K ตัวถัง CABGA256):**
    ```bash
    make DEVICE=12k PACKAGE=CABGA256
    ```

### ⏱️ การจำกัดเวลาโปรแกรมจำลองสถานะวงจร (Timeout Protection)
เพื่อป้องกันเครื่องค้างจาก Loop อนันต์ ระบบจะจำลองเวลาไว้ที่ `1ms` เป็นค่าเริ่มต้น แต่คุณสามารถสั่งตัดเวลาหยุดรันได้ตามต้องการ:
```bash
make sim SIM_TIME=500us
make wave SIM_TIME=10ms
```

## 💻 วิธีการรันโปรเจกต์แบบแยกขั้นตอนย่อย (Granular Design Flow)

สามารถเลือกสั่งงานใน `Makefile` แยกทีละกระบวนการ โดยผลลัพธ์ทั้งหมดจะถูกเรียกประมวลผลอยู่ด้านในโฟลเดอร์ `build/`:

### 1. สังเคราะห์วงจร (Synthesis)
```bash
make synth
```
*(คอมไพล์โค้ดภาษา VHDL ออกเป็นไฟล์โครงสร้างวงจรดิจิทัลรูปแบบ JSON ไว้ที่พาธ `build/blinky.json`)*

### 2. วาดภาพผังวงจร (Render Schematic)
```bash
make view
```
*(สร้างไฟล์ภาพผังวงจรดิจิทัลรูปไฟล์ `build/schematic.svg` อัตโนมัติด้วยคำสั่งเครื่องมือ **netlistsvg**)*

### 3. จำลองคลื่นสัญญาณ (Simulation & GTKWave)
```bash
make wave
```
*(ประมวลผลคัดแยกข้อมูลเวลาลอจิกออกเป็นไฟล์คลื่น `build/waveform.vcd` แล้วเรียกเปิดหน้าจอแอปพลิเคชัน GUI ของ **GTKWave** ทันที)*

### 4. การดีบั๊กซอร์สโค้ดข้ามภาษาผ่าน GDB (GNU Debugger)
เมื่อมีการเชื่อมโยงโค้ดฟังก์ชันภาษา C++ มาร่วมรันผ่านสิทธิ์อินเทอร์เฟส **VHPIDirect** คุณสามารถเปิดใช้งาน GDB เพื่อตรวจสอบค่าตัวแปรข้ามภาษาบรรทัดต่อบรรทัดได้ทันที:
```bash
make debug
```

### 5. จัดวางและหาเส้นทางวงจร (Place & Route)
```bash
make pnr
```
### 6. บีบอัดไฟล์พร้อมแฟลช (Pack Bitstream)
```bash
make pack
```
### 7. ล้างไฟล์เอาต์พุตทั้งหมด (Clean Workspace)
```bash
make clean
```

> 💡 **ข้อควรระวังสำหรับ GTKWave GUI บน WSL:** ตัวระบบต้องทำงานอยู่บน **WSL 2 บนระบบปฏิบัติการ Windows 11** ขึ้นไปเพื่อเปิดแอปพลิเคชัน Linux GUI ทันทีแบบอัตโนมัติ แต่หากคุณใช้งานอยู่บน **Windows 10** คุณจำเป็นต้องติดตั้งโปรแกรมเซิร์ฟเวอร์ช่วยแสดงผลเพิ่มเติม เช่น [VcXsrv](https://vcxsrv.com/) บนระบบ Windows ตัวหลักล่วงหน้า
